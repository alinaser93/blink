عدّل **طبقة البيانات** — وهي المصدر المشترك الذي يربط كل الصفحات.

الملفات المستهدفة:
- `src/lib/api.js` — دوال القراءة/الكتابة (تعمل في وضعَي live + تجريبي via `isLive`).
- `src/lib/supabase.js`, `src/lib/mockData.js` — الاتصال والبيانات التجريبية.
- `supabase/schema.sql`, `supabase/seed.sql` — سكيمة قاعدة البيانات (وثّق أي تغيير هنا).

التعليمات:
1. اقرأ الملفات المستهدفة بالكامل، واقرأ `CLAUDE.md` للسياق.
2. **أي تعديل على سلوك البيانات = دالة في `api.js`** تعمل live و mock، **+ تعديل/توثيق السكيمة** في `supabase/schema.sql` (آمن لإعادة التشغيل: `if not exists` / `do $$ ... $$`).
3. حافظ على تطابق المنطق بين الوضعين (live و mock)، وعلى أمان RLS عند اللزوم.
4. تذكّر: هذا الملف هو **نقطة الترابط** — الأدمن والمنتجات والأقسام والمندوب كلهم يستهلكونه.
5. بعد التعديل: `npm run build` وتأكّد لا أخطاء.
6. لا تدفع ولا تنشر على `main` إلا حين أطلب صراحةً.
