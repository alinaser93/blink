# سلّـة / اطلبها — تطبيق كويك-كومرس بنمط Blinkit

> **الشمال الحقيقي:** كل صفحة يجب أن تُطوَّر لتشبه تطبيق **Blinkit** — نظيفة، سريعة، بصرية.

## الهوية والقواعد الثابتة
- اللغة: **عربي (لهجة عراقية ودّية)** · الاتجاه: **RTL** · العملة: **دينار عراقي «د.ع»** · الخط: **Cairo**.
- اللون الأساسي **أخضر `#0C831F`**، وذهبي `#D4AF37` للعروض/الشارات.
- **خصائص CSS منطقية فقط** (`inset-inline`, `ms-/me-`, `text-start/end`) — تجنّب الفيزيائية (`ml-/mr-/text-left`).
- **صور المنتجات = بلاطات إيموجي ملوّنة** عبر `emojiFor()` من `src/customer/emoji.js` (لا توجد صور حقيقية).
- لا `localStorage`. صيغة السعر عبر `iqd()`.

## الستاك والنشر
- Vite + React + Tailwind + react-router + Supabase. النشر على **Netlify**.
- **النشر = دفع على فرع `main`** → Netlify يبني وينشر تلقائياً (~1-2 دقيقة).
- التطوير على فرع `claude/...` ثم الدمج على `main` عند الطلب فقط.
- بعد كل تعديل: `npm run build` والتأكّد من عدم وجود أخطاء.

## كيف تترابط الأجزاء (مهم)
التكامل عبر **الكود وقاعدة البيانات المشتركة**، لا عبر المحادثات:
- إضافة **صفحة جديدة** = ملف جديد + `Route` في `src/App.jsx` + رابط/زر يوصّلها.
- **الأدمن ↔ المنتجات/الأقسام**: كلاهما يقرأ/يكتب عبر `src/lib/api.js` + جداول Supabase.
- **المندوب ↔ الأدمن**: المندوب يحدّث حالة الطلب (`setOrderStatus` → جدول `orders`)، واللوحة تقرأ نفس الحالة (السكيمة مجهّزة لـ Realtime). فالتواصل = **حالة الطلب في قاعدة البيانات**.

## نموذج العمل
كتالوج موحّد بنمط Blinkit (Dark Store). الأدوار: **customer / admin / rider** فقط — **لا يوجد دور «تاجر»** (لوحة «التاجر» هي فعلياً لوحة الأدمن/مدير الفرع). أي بوابة تاجر حقيقية = إضافة معمارية كبيرة تحتاج موافقة.

## وضع البيانات
`isLive` في `src/lib/supabase.js`: إذا ضُبطت متغيّرات `VITE_SUPABASE_*` يعمل على قاعدة حيّة، وإلا على **بيانات تجريبية في الذاكرة** (تنضبط بالجلسة). الواجهات يجب أن تعمل في الوضعين.
